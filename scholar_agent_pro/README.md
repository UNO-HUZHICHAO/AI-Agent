# ScholarAgent Pro 技术说明文档

## 1. 技术架构

### 1.1 系统整体架构

ScholarAgent Pro 是一个基于 Streamlit 的学术文献数据检索与趋势分析智能体，采用模块化设计，主要包括以下核心模块：

```
┌─────────────────────────────────────────────────────────────────┐
│                     ScholarAgent Pro                            │
├───────────┬─────────────────┬─────────────────┬─────────────────┤
│ 基础配置  │   爬虫模块      │   计量分析模块  │    AI分析与报告模块 │
├───────────┼─────────────────┼─────────────────┼─────────────────┤
│ 页面配置  │ CNKI爬虫       │ 时间趋势回归   │ 关键词分析      │
│ 期刊白名单 │ OpenAlex爬虫   │ 关键词-被引数回归 │ 关键词翻译      │
│ 工具函数  │ 详情页抓取     │ 预测模型       │ 智能综述生成    │
│           │ 数据清洗       │ 高影响力关键词分析 │ 思维导图生成    │
├───────────┼─────────────────┼─────────────────┼─────────────────┤
│           │               主界面模块                          │
├───────────┼─────────────────┼─────────────────┼─────────────────┤
│           │ Streamlit UI   │ 会话状态管理   │ 数据可视化      │
└───────────┴─────────────────┴─────────────────┴─────────────────┘
```

### 1.2 数据流图

```
用户输入研究描述 → 关键词提取与翻译 → 多数据源抓取 → 数据清洗与整合 → 计量分析 → AI报告生成 → 结果展示
```

1. **用户输入**：用户在界面上输入研究问题描述，设置API密钥和抓取参数
2. **关键词分析**：使用LLM从研究描述中提取中文关键词，并自动翻译为英文关键词
3. **多数据源抓取**：同时抓取CNKI（中文）和OpenAlex（英文）的学术文献数据
4. **数据清洗与整合**：去重、格式统一、期刊等级判断、中英文数据整合
5. **计量分析**：进行时间-文献数量趋势回归和关键词-被引数相关性分析
6. **AI报告生成**：基于分析结果生成结构化的学术综述和HTML格式思维导图
7. **结果展示**：通过Streamlit界面展示分析结果、图表和报告

## 2. 计量方法原理

### 2.1 时间趋势回归模型

**模型类型**：一元线性回归模型

**模型公式**：
```
Count = β₀ + β₁ × Year + ε
```

**变量解释**：
- `Count`：每年发表的文献数量
- `Year`：发表年份
- `β₀`：截距项
- `β₁`：斜率项，表示文献数量随年份的变化趋势
- `ε`：随机误差项

**模型评估指标**：
- **R²**：拟合优度，衡量模型对数据的解释能力，取值范围[0,1]
- **P值**：显著性水平，用于检验自变量对因变量的影响是否显著，通常P<0.05表示显著
- **斜率**：表示每年文献数量的变化率

**预测功能**：基于回归模型预测未来5年的文献数量变化趋势

### 2.2 关键词-被引数相关性回归模型

**模型类型**：一元线性回归模型

**模型公式**：
```
Citations = β₀ + β₁ × Frequency + ε
```

**变量解释**：
- `Citations`：文献被引次数
- `Frequency`：关键词出现频率
- `β₀`：截距项
- `β₁`：斜率项，表示关键词频率对被引次数的影响程度
- `ε`：随机误差项

**模型评估指标**：
- **R²**：拟合优度，衡量模型对数据的解释能力
- **P值**：显著性水平，用于检验关键词频率对被引次数的影响是否显著
- **斜率**：表示关键词频率每增加1单位，被引次数的变化量

**高影响力关键词分析**：计算每个关键词的平均被引次数，识别领域内的高影响力关键词

## 4. 部署步骤

### 4.1 本地部署

1. **准备代码**：
   将代码文件复制到本地目录
   ```bash
   cd /path/to/your/directory
   ```

2. **安装依赖**：
   ```bash
   pip install -r requirements.txt
   ```

3. **运行应用**：
   ```bash
   streamlit run scholar_agent_pro.py
   ```

4. **访问应用**：
   在浏览器中打开输出的URL（通常是 http://localhost:8501）

### 4.2 Docker部署（推荐）

1. **创建Dockerfile**：
   ```dockerfile
   FROM python:3.11-slim
   
   WORKDIR /app
   
   COPY requirements.txt .
   RUN pip install --no-cache-dir -r requirements.txt
   
   COPY . .
   
   EXPOSE 8501
   
   CMD ["streamlit", "run", "scholar_agent_pro.py"]
   ```

2. **构建Docker镜像**：
   ```bash
   docker build -t scholar-agent-pro .
   ```

3. **运行Docker容器**：
   ```bash
   docker run -p 8501:8501 scholar-agent-pro
   ```

4. **访问应用**：
   在浏览器中打开 http://localhost:8501

### 4.3 常见问题排查

1. **浏览器启动失败**：
   - 检查Chrome是否已安装
   - 尝试手动关闭旧的Chrome窗口
   - 检查是否有足够的系统资源

2. **API调用失败**：
   - 检查API Key是否正确
   - 检查网络连接
   - 检查API服务是否正常

3. **回归分析失败**：
   - 检查数据是否足够（至少包含2年的数据）
   - 检查数据格式是否正确
   - 检查是否有异常值

4. **知网抓取失败**：
   - 可能触发了知网的滑块验证，需要手动完成
   - 检查网络连接是否稳定
   - 尝试减少抓取数量

## 5. API接口说明

### 5.1 LLM API配置

ScholarAgent Pro 使用 DeepSeek API 进行关键词分析、翻译和报告生成。

**配置参数**：
- **API Key**：DeepSeek API密钥
- **API Base URL**：API基础URL，默认值为 `https://api.deepseek.com`

**模型选择**：
- 默认使用 `deepseek-chat` 模型
- 支持调整 temperature 参数控制生成文本的随机性

### 5.2 爬虫接口

#### 5.2.1 CNKI爬虫

**功能**：爬取中国知网的学术文献数据

**参数**：
- `topic`：研究主题
- `target_count`：目标抓取数量
- `strategy`：抓取策略（HYBRID/SC/SU）
- `log_container`：日志容器
- `fetch_details`：是否抓取详情页（摘要+关键词）
- `keywords`：可选，用于增强搜索的关键词列表

**返回值**：
- 包含文献信息的DataFrame，字段包括标题、来源、年份、被引次数、关键词、摘要、期刊等级、抓取策略、语言等

#### 5.2.2 OpenAlex爬虫

**功能**：爬取OpenAlex的英文文献数据

**参数**：
- `topic`：研究主题
- `target_count`：目标抓取数量
- `log_container`：日志容器
- `keywords`：可选，用于增强搜索的关键词列表

**返回值**：
- 包含文献信息的DataFrame，字段与CNKI爬虫类似

### 5.3 计量分析接口

#### 5.3.1 时间趋势回归

**功能**：进行时间-文献数量趋势回归分析

**参数**：
- `df`：文献数据DataFrame

**返回值**：
- 包含回归结果的字典，包括R²、P值、斜率、截距、回归方程、预测数据等

#### 5.3.2 关键词-被引数相关性回归

**功能**：进行关键词频率与被引次数的相关性分析

**参数**：
- `df`：文献数据DataFrame

**返回值**：
- 包含回归结果的字典，包括R²、P值、斜率、截距、回归方程、分析数据等

### 5.4 AI分析与报告接口

#### 5.4.1 关键词分析

**功能**：从研究描述中提取关键词

**参数**：
- `topic`：研究主题
- `api_key`：DeepSeek API Key
- `base_url`：API基础URL
- `language`：主题语言（zh或en）

**返回值**：
- 提取的关键词列表（恰好5个关键词）

#### 5.4.2 关键词翻译

**功能**：将关键词翻译成目标语言

**参数**：
- `keywords`：关键词列表
- `target_lang`：目标语言（en或zh）
- `api_key`：DeepSeek API Key
- `base_url`：API基础URL

**返回值**：
- 翻译后的关键词列表

#### 5.4.3 智能综述生成

**功能**：生成结构化的学术综述报告

**参数**：
- `df`：文献数据DataFrame
- `topic_cn`：中文研究主题
- `topic_en`：英文研究主题
- `api_key`：DeepSeek API Key
- `base_url`：API基础URL

**返回值**：
- 流式响应对象（用于实时显示报告）和思维导图HTML文本

#### 5.4.4 思维导图生成

**功能**：生成HTML格式的思维导图

**参数**：
- `topic`：研究主题
- `df`：文献数据DataFrame
- `time_regression_res`：时间回归分析结果
- `kw_regression_res`：关键词回归分析结果

**返回值**：
- HTML格式的思维导图文本

## 6. 使用指南

### 6.1 功能模块介绍

#### 6.1.1 数据抓取

- **研究问题描述**：输入详细的研究问题描述，系统会从中提取关键词
- **关键词分析**：自动从研究描述中提取中文关键词，并翻译为英文关键词
- **关键词编辑**：支持手动调整生成的关键词，提高检索精准度
- **抓取设置**：设置每个数据源的抓取数量、是否抓取详情页
- **抓取策略**：选择CNKI抓取策略（混合均衡模式、高被引优先、相关度优先）
- **数据源选择**：同时选择中文文献（CNKI）和英文文献（OpenAlex）

#### 6.1.2 计量分析

- **时间-文献数量回归**：展示文献数量随时间的变化趋势和回归结果，包括R²、P值、斜率等指标
- **关键词-被引数相关性**：分析关键词频率与被引次数的关系，展示相关性散点图和回归线
- **高影响力关键词**：展示平均被引次数较高的关键词，识别领域内的研究热点

#### 6.1.3 智能综述

- **生成智能综述报告**：基于分析结果生成结构化的学术综述，包括研究背景、核心方向、时间分布、高被引文献分析等
- **思维导图**：生成研究领域的HTML格式思维导图，支持折叠/展开节点，具有更好的可视化效果
- **报告下载**：支持下载HTML格式的思维导图，可直接在浏览器中打开查看

### 6.2 操作步骤

1. **配置参数**：
   - 在左侧侧边栏输入详细的研究问题描述
   - 输入DeepSeek API Key和API Base URL
   - 设置每个数据源的抓取数量和是否抓取详情页
   - 选择CNKI抓取策略

2. **提取关键词**：
   - 点击"从描述中提取关键词"按钮
   - 系统会自动从研究描述中提取中文关键词并翻译为英文关键词
   - 查看并根据需要调整生成的关键词

3. **抓取数据**：
   - 点击"开始自动抓取"按钮
   - 系统会同时抓取CNKI和OpenAlex的文献数据
   - 等待抓取完成（可能需要手动完成知网的滑块验证）

4. **查看分析结果**：
   - 切换到"计量分析"标签页查看回归结果和图表
   - 切换到"智能综述"标签页生成并查看AI报告
   - 下载生成的思维导图用于后续研究

### 6.3 最佳实践

1. **关键词优化**：
   - 输入详细的研究问题描述，系统会提取更准确的关键词
   - 分析生成的关键词后，手动调整以提高检索精准度
   - 保留最相关的5个关键词

2. **抓取策略选择**：
   - 初次检索建议使用"混合均衡模式"
   - 关注经典文献使用"高被引优先"
   - 关注最新研究使用"相关度优先"

3. **数据量选择**：
   - 建议每个数据源抓取15-30篇文献
   - 过少的数据可能导致回归分析不准确
   - 过多的数据会增加抓取时间和系统负担

4. **报告生成**：
   - 确保输入有效的DeepSeek API Key
   - 等待报告生成完成，不要中途刷新页面
   - 可以下载思维导图用于后续研究和展示

5. **知网抓取技巧**：
   - 抓取过程中可能会触发知网的滑块验证，需要及时手动完成
   - 建议在网络稳定的环境下进行抓取
   - 避免短时间内频繁抓取，以免被知网限制访问
